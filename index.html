Selection rule

  4. Treat selected variables as KPIs.
Judgment: The 85% rule keeps most predictive information while reducing model complexity.

Q3 KPI Selection

Q3 · How to Read the Evaluation Metrics

Recall

TP / (TP + FN)

Among actual subscribers, how many did the model find?

Precision

TP / (TP + FP)

Among predicted subscribers, how many were truly subscribers?

F1-score

2PR / (P + R)

Balance between recall and precision.

Accuracy

Correct / Total

Overall correctness, but can be misleading under imbalance.

Q3 judgment priority: Since yes is rare but valuable, recall and F1-score are more meaningful than accuracy alone.

Q4 · Why XGBoost Fails in This Output

Observed result

  • XGBoost recall = 0.
  • Precision and F1-score are NA.
  • The model fails to identify any Vol customer.
Important: This does not mean XGBoost is a weak algorithm. It means this specific tuning result did not train the model properly.

Reasoning logic

XGBoost depends on sequential boosting rounds. Here, the selected setting is:

nrounds = 1
  • Only one tree is trained.
  • The boosting mechanism is almost absent.
  • The model becomes under-trained.
  • It predicts the majority class and misses Vol churners.
